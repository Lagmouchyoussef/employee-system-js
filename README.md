## üåü About

This repository contains a series of hands-on exercises designed to solidify your understanding of JavaScript fundamentals, with a strong focus on objects. By modeling a real-world HR scenario for the fictional company "TechCorp", you will learn to create, manipulate, and secure data, as well as automate common business tasks.

## üéØ Learning Objectives

By completing these exercises, you will master:
*   **Object Creation & Manipulation:** Creating, reading, updating, and deleting (CRUD) object properties.
*   **Nested Objects:** Managing complex data structures like addresses and contacts.
*   **Methods:** Encapsulating behavior within objects.
*   **Data Validation:** Implementing business rules and constraints.
*   **Arrays & Collections:** Working with lists of employees and departments.
*   **Advanced Logic:** Implementing search, sorting, and audit logging.

## ‚ú® Features

The HR module you will build includes:
*   **Employee Management:** Create and update employee profiles.
*   **Department Organization:** Structure employees into departments with budgets.
*   **Statistical Reporting:** Calculate payroll, headcount, and average salaries.
*   **Leave Management:** A system for employees to request and track their vacation days.
*   **Audit Trail:** A log to track critical changes like salary updates.

## üè¢ Project Context

*   **Company:** TechCorp, a leader in digital services.
*   **Your Role:** Junior Developer.
*   **Mission:** Develop the first version of an internal personnel management module using JavaScript in console mode.

## üìö Exercises

The project is divided into several parts, each focusing on a specific set of skills.

### Part 1 - Basic Employee Modeling
*   **Exercise 1:** Create an employee object.
*   **Exercise 2:** Modify properties and track changes.
*   **Exercise 3:** Delete obsolete properties.

### Part 2 - Nested Objects & HR Relations
*   **Exercise 4:** Add nested objects for `address` and `emergencyContact`.
*   **Exercise 5:** Iterate over all properties of an object.

### Part 3 - Collective Management: Departments
*   **Exercise 6:** Create a department object containing an array of employees and a method to calculate the average salary.

### Part 4 - Security & Validation
*   **Exercise 8:** Write a function to check if an employee record is compliant with HR rules.

### Part 5 - HR Statistics & Reports
*   **Exercise 10:** Generate global statistics from an array of employees (active count, total payroll, average salary by department).

### Part 6 - Leave Management, Search & Sorting
*   **Exercise 11:** Implement a `takeVacation()` method for employees.
*   **Exercise 12:** Create a generic `findBy()` search function.
*   **Exercise 13:** Sort an array of employees by multiple criteria.

### Part 7 - Budgets & Grouping
*   **Exercise 14:** Add a `spend()` method to department objects for budget management.
*   **Exercise 15:** Group employees by department to get headcount and average salary.

### Part 8 - Audit & Reporting
*   **Exercise 16:** Implement an audit log to track all salary changes.

## üöÄ Getting Started

Follow these instructions to set up the project locally and start coding.

1.  **Clone the repository**
    ```bash
    git clone https://github.com/Lagmouchyoussef/employee-system-js.git
    ```

2.  **Navigate to the project directory**
    ```bash
    cd employee-system-js
    ```

3.  **Open the project in your code editor** (e.g., VS Code).

4.  **Start coding!** Open the `script.js` file and implement the exercises one by one.

5.  **Run your code** using Node.js:
    ```bash
    node script.js
    ```
